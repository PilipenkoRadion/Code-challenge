🔄 Systemarchitektur

Client (HTML/CSS/JS) ↓ POST-Anfrage mit Code Frontend (/level/) ← Django Views ↓ Django Views (select_level, send_task) ├─ get_challenge() → Aufgabe abrufen ├─ run_and_test() → Code testen └─ run_[language]() → In gewünschter Sprache ausführen ↓ Subprocess (Code-Ausführung) ├─ Python: sys.executable -c ├─ JavaScript: node (Temp-Datei) ├─ Java: javac + java (Temp-Verzeichnis) ├─ C++: g++ (Temp-Verzeichnis) └─ Go: go build (Temp-Verzeichnis) ↓ Test Cases Validation ├─ Output mit expected_output vergleichen ├─ Alle Tests überprüfen └─ Ergebnisse zurückgeben ↓ Response JSON └─ all_passed, passed/total, Ergebnisse jedes Tests

📊 Technische Spezifikationen

⏱️ Timeouts

• Kompilation: 10 Sek
• Ausführung: 5 Sek
• Request-Verarbeitung: asynchron

📋 Aufgaben

• Easy: 10 Aufgaben (10 Punkte)
• Medium: 10 Aufgaben (20-25 Punkte)
• Hard: 10 Aufgaben (30-50 Punkte)

🛡️ Sicherheit

• Verwendung von subprocess.run
• Temp-Verzeichnisse zur Isolation
• Automatisches Löschen von Dateien

🔤 Sprachen

• Python 3.8+
• JavaScript (Node.js)
• Java, C++, Go

🗺️ Views und Routen

View/URL Methode Funktion
index() GET Startseite mit Gradient und Information
select_level() POST Auswahl des Schwierigkeitsgrades (easy, medium, hard)
select_language() POST Auswahl der Programmiersprache
main_task_page() GET Hauptseite mit Code-Editor und Testung
